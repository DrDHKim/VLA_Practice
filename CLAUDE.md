# CLAUDE.md — Claude Code 작업 규칙

이 파일은 Claude Code가 세션 시작 시 읽는 프로젝트별 규칙이다. 구현 지시서가 아니라 작업 방식과 커밋 절차를 정의한다. 구현 순서는 `TASKS.md`, 프로젝트 방향은 `AGENTS.md`를 따른다.

## 커밋 절차

커밋은 아래 순서를 반드시 지킨다.

### 1. 연구일지 최신화

`docs/research_journal.md`를 읽고 `git diff`와 비교해 누락된 항목이 있으면 먼저 기록한다. 커밋 내용과 일지가 일치해야 한다.

### 2. 미추적 파일 처리

`git status`로 untracked 파일을 확인한다. 파일마다 아래 중 하나를 결정하고, 결정 전까지 커밋하지 않는다.

- **추적**: 코드·문서·설정·테스트로 저장소에 포함해야 하는 파일
- **gitignore 추가**: 런타임 산출물·캐시·대용량 바이너리
- **삭제**: 더 이상 필요 없는 실험용 임시 파일

파일별 추적 여부가 불명확하면 사용자에게 확인한다. unstaged 파일이 남아 있는 상태로 커밋하지 않는다.

### 3. diff 전수 검토

`git diff HEAD --stat`과 파일별 diff를 모두 읽는다. 변경 내용이 커밋 메세지에 반영됐는지 확인한다.

### 4. 커밋 메세지 제안 전 최종 점검

아래 세 가지를 모두 확인한 뒤에만 커밋 메세지를 사용자에게 제안한다.

- `git status`에서 untracked/unstaged 파일이 없다.
- diff와 연구일지에서 확인한 변경 사항이 커밋 메세지에 포함됐다.
- diff나 일지에서 봤지만 메세지에서 빠진 항목이 없다.

### 5. 미검증 기능은 커밋 제외

신규 기능을 추가했더라도 smoke run이나 unit test로 검증되기 전에는 커밋에 포함하지 않는다. 검증이 끝난 기능만 묶어서 커밋한다.

### 6. 커밋 메세지 제안

위 점검이 끝나면 커밋 메세지를 사용자에게 제안한다. 사용자가 확인하면 그때 `git commit`을 실행한다.

## 코드 규칙

`AGENTS.md`의 코드 규칙과 동일하게 따른다.

- `src/vla_drive/` 패키지 구조를 유지한다.
- 새 파일보다 기존 stub 파일을 먼저 확인한다.
- 폴더 구조는 `TASKS.md`가 요구할 때만 바꾼다.
